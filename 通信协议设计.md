# 机房巡检小车简化通信协议

## 1. 串口分工

| 串口 | 对象 | 波特率 | 用途 |
|---|---|---:|---|
| USART1 | 树莓派 | 921600 | 状态、温湿度、电池、热成像、命令 |
| USART2 | 八路巡线模块 | 115200 | 巡线模块数据 |

USART1 建议 DMA 发送 + DMA 接收。USART2 后续也尽量 DMA 接收。

## 2. 帧格式

第一版用极简帧：

```text
0x55 + msg_id + len + payload + checksum
```

字段：

| 字段 | 长度 | 说明 |
|---|---:|---|
| head | 1 | 固定 `0x55` |
| msg_id | 1 | 消息 ID |
| len | 1 | payload 长度，0~255 |
| payload | len | 数据 |
| checksum | 1 | 和校验 |

和校验：

```c
checksum = msg_id + len + payload[0] + ... + payload[len - 1];
```

只取低 8 位：

```c
uint8_t checksum;
```

帧头 `0x55` 不参与校验。

为什么保留 `len`：

- 没有 `len`，接收端不知道一包多长。
- 热成像要分片，必须知道每片长度。
- 只多 1 字节，但稳定很多。

## 3. 接收解析

状态机：

```text
WAIT_HEAD
READ_MSG_ID
READ_LEN
READ_PAYLOAD
READ_CHECKSUM
CHECK
DISPATCH
```

异常处理：

- 帧头不是 `0x55`：继续等帧头。
- `len` 超过最大值：丢包。
- 和校验错误：丢包。

建议最大 payload：

```c
#define PROTOCOL_MAX_PAYLOAD 80
```

80 字节足够发热成像一行。

## 4. 消息 ID

| ID | 方向 | 名称 | 周期 |
|---:|---|---|---:|
| `0x01` | STM32 -> 树莓派 | 小车状态 | 100ms |
| `0x02` | STM32 -> 树莓派 | 温湿度 + 电池 | 100ms/500ms |
| `0x03` | STM32 -> 树莓派 | 巡线状态 | 100ms |
| `0x04` | STM32 -> 树莓派 | 热成像摘要 | 100ms |
| `0x05` | STM32 -> 树莓派 | 热成像行数据 | 有新帧就发 |
| `0x10` | 树莓派 -> STM32 | 控制命令 | 按需 |
| `0x11` | 树莓派 -> STM32 | 参数设置 | 按需 |
| `0x12` | STM32 -> 树莓派 | 异常事件 | 事件触发 |

## 5. 数据约定

不发 `float`，全部用整数。

| 数据 | 单位 | 示例 |
|---|---:|---:|
| 电压 | mV | `7420` = 7.420V |
| 温度 | 0.01 degC | `2534` = 25.34 degC |
| 湿度 | 0.01 %RH | `6012` = 60.12% |
| 速度 | cm/s | `18` = 18cm/s |
| 角度 | 0.01 deg | `9012` = 90.12deg |
| 热成像 raw | 0.02 degC | `1250` = 25.00degC |

多字节数据统一小端序。

## 6. Payload 定义

### 6.1 小车状态 `0x01`

周期：100ms。

```c
typedef struct {
    uint32_t tick_ms;
    uint8_t  state;
    uint8_t  fault;
    uint8_t  lap_count;
    uint8_t  battery_percent;
    int16_t  speed_l_cms;
    int16_t  speed_r_cms;
    int16_t  yaw_cdeg;
} CarStatusPayload_t;
```

长度：16 字节。

### 6.2 温湿度 + 电池 `0x02`

周期：100ms 或 500ms。

```c
typedef struct {
    uint32_t tick_ms;
    int16_t  temp_centi;
    int16_t  humi_centi;
    uint16_t battery_mv;
    uint8_t  battery_percent;
    uint8_t  power_state;
} EnvBatteryPayload_t;
```

长度：12 字节。

### 6.3 巡线状态 `0x03`

周期：100ms。

```c
typedef struct {
    uint32_t tick_ms;
    int16_t  track_error;
    uint8_t  track_ready;
    uint8_t  use_analog;
    uint16_t analog[8];
} TrackStatusPayload_t;
```

长度：24 字节。

### 6.4 热成像摘要 `0x04`

周期：100ms。

```c
typedef struct {
    uint32_t tick_ms;
    uint16_t frame_id;
    int16_t  max_temp_centi;
    int16_t  min_temp_centi;
    int16_t  avg_temp_centi;
    uint8_t  max_x;
    uint8_t  max_y;
} ThermalSummaryPayload_t;
```

长度：14 字节。

### 6.5 热成像行数据 `0x05`

MLX90642 是 32x24。每帧拆 24 包，每包一行。

```c
typedef struct {
    uint32_t tick_ms;
    uint16_t frame_id;
    uint8_t  row;          // 0~23
    uint8_t  width;        // 32
    uint16_t temp_raw[32]; // 每点温度 = raw * 0.02
} ThermalRowPayload_t;
```

长度：72 字节。

树莓派按 `frame_id + row` 拼成完整热图。

### 6.6 控制命令 `0x10`

```c
typedef struct {
    uint8_t cmd;
    int16_t arg1;
    int16_t arg2;
} CommandPayload_t;
```

命令：

```c
0 STOP
1 START_PATROL
2 PAUSE_PATROL
3 RESUME_PATROL
4 SET_SPEED
5 TURN_ANGLE
6 ENTER_LOW_POWER
```

### 6.7 参数设置 `0x11`

```c
typedef struct {
    uint8_t param_id;
    int16_t value;
} ParamSetPayload_t;
```

参数：

```c
1 TRACK_KP
2 TRACK_KD
3 BASE_SPEED
4 LOW_BAT_PERCENT
5 END_BOX_CONFIRM_MS
```

### 6.8 异常事件 `0x12`

```c
typedef struct {
    uint32_t tick_ms;
    uint8_t  fault_code;
    uint16_t detail;
} FaultEventPayload_t;
```

异常码：

```c
1 TRACK_LOST
2 BATTERY_LOW
3 MLX90642_FAIL
4 SHT31_FAIL
5 USART_OVERFLOW
6 MOTOR_STALL
```

## 7. 热成像带宽

一行热成像：

```text
payload 72 字节
整帧 24 行
约 24 * (1+1+1+72+1) = 1824 字节
8Hz 约 14.6KB/s
```

USART1 用 921600 足够。

115200 不够稳定传全帧热成像。

## 8. 第一版实现范围

先做这些：

1. USART1 改 921600。
2. USART1 DMA 发送。
3. USART1 DMA 接收 + IDLE。
4. 实现 `Protocol_SendFrame(msg_id, payload, len)`。
5. 实现接收解析状态机。
6. 发送 `0x01/0x02/0x03`。
7. 再发送 `0x04/0x05` 热成像。

USART2：

1. 先保留当前 `$0,1,1#`。
2. 后续改 DMA 接收。

## 9. 示例帧

发送小车状态，`msg_id=0x01`，payload 长度 16：

```text
55 01 10 [16字节payload] checksum
```

发送热成像第 3 行，`msg_id=0x05`，payload 长度 72：

```text
55 05 48 [72字节payload] checksum
```

## 10. 备注

- 简单协议，不做 ACK。
- 树莓派如果发现丢行，直接丢弃当前热成像帧。
- STM32 只负责持续发送。
- 后续如果现场干扰大，再升级 CRC16。
